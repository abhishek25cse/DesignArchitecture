- **Structural Patterns**: Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
- **Behavioral Patterns**: Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor, Interpreter

**Key Resources:**
- [Pattern Comparison Guide](./DesignPattern/PATTERN_COMPARISON.md)

### 2. [SOLID Principles](./Principles/README.md)
In-depth exploration of the five SOLID principles for object-oriented design.

**What you'll learn:**
- **S**ingle Responsibility Principle (SRP)
- **O**pen/Closed Principle (OCP)
- **L**iskov Substitution Principle (LSP)
- **I**nterface Segregation Principle (ISP)
- **D**ependency Inversion Principle (DIP)

**Key Resources:**
- [SOLID Overview](./Principles/SOLID_OVERVIEW.md) - Comprehensive guide with history and real-world examples
- [SOLID Summary](./Principles/SOLID_PRINCIPLES_SUMMARY.md) - Quick reference with all principles in one place
- Individual principle documents with extensive examples and diagrams

## 🎯 Learning Paths

### For Beginners
1. Start with **SOLID Principles** - Foundation of good design
   - Begin with Single Responsibility Principle
   - Move to Interface Segregation Principle
   - Learn Dependency Inversion Principle
2. Learn basic **Design Patterns**
   - Singleton, Factory Method, Observer
3. Study **Decision Making** fundamentals

### For Intermediate Developers
1. Deep dive into **Design Patterns**
   - Decorator, Strategy, Adapter
   - Command, State, Template Method
2. Master all **SOLID Principles**
3. Apply patterns to real projects
4. Study **Language Comparison** for architecture decisions

### For Advanced Developers
1. Master complex **Design Patterns**
   - Visitor, Interpreter, Flyweight
   - Mediator, Memento, Chain of Responsibility
2. Understand pattern relationships and combinations
3. Learn when NOT to use patterns (Anti-Patterns)
4. Make informed architectural decisions using decision frameworks
